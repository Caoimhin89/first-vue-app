Reacting to Changes in Computed Properties

Counter is incremented or decremented here through use of Vue.js's Methods and Computed properties. Methods execute whenever a DOM element is changed, while Computed functions will execute only when an element it depends on is changed. By opening the JavaScript console in Chrome's dev tools, you can see this in action. Whenever you increase or decrease the counter, both our Method & Computed functions will execute and a corresponding message will be printed to the console. However, when you increment the Second Counter, only our Method function will execute and print its message to the console. This is because our Second Counter is not connected to our Result or Output, only the Counter is. Counter changes our Output, which is the Computed property, while Second Counter does not change Output. Therefore, our Computed Property will not be updated. Result is our Methods property, and although it isn't connected with Second Counter either, Methods properties will always update anytime a DOM element is updated. So you will still see our Methods property go to work and print a message to the console when you increment the Second Counter.

Watch

Computed properties are synchronous. They have to be, because their job is to update immediately whenever a DOM element is changed. However, sometimes you need to perform asynchronous tasks (requesting data from a server and updating the DOM once the server responds with the requested data, for example). This is where Vue.js's Watch property comes into play. It will watch for a certain operation to complete and update the DOM once it is. We demonstrate this here with a setTimeout of 5 seconds on a Watch function that resets our counter to zero. So you'll notice that after waiting 5 seconds after incrementing or decrementing the counter, it will be reset to zero. Give it a try!
